The future of web payments in Google IO 2017.

未来的 WEB 支付方式 - 2017 Google IO

Posted by mslinzz on 2017-06-20

前言

What's new in Google IO 2017. 每年被新技术的洗礼, 被全球各地的开发爱好者期待着的 Google IO 大会在 May 17 - 19, 美国加州举行了.

我也曾几次跟着朋友们去参加北京线下组织的 Google IO 大会, 欣赏欣赏美景. 这次我尊敬的leader赴美归来, 想把这次看到的关于 WEB 的新元素分享给我们团队, 为了大家充分受益, 特地选了 20+ 关于 WEB 的 Toutube 视频, 让我们选择自己感兴趣的主题进行观看, 然后给大家分享.

我选择的其中一个 Event 是 The future of web payments.

先来介绍一下演讲者:

Zach Koch Google Chrome Product Manager

Zach is trying to save the world from annoying checkout forms.

Zach 做了一些调查, 在国外, 使用浏览器去购物还是比较常见的, 流行很多商家自己的web商店, 当用户每次进行购物, 点击购买的一瞬间就去跳转到了录入各类信息的表单, 有些用户因为麻烦, 操作频繁等放弃这次交易. 因此商家必定会损失一大批用户.

调查显示, 虽然移动流量在不断增长, 但移动购物率仅占所有已完成购物活动的三分之一. 换言之, 移动设备用户的购物放弃率是桌面设备用户的两倍.

此次, Zach 提到, 在用户支付时只关心点击购买和确认支付即可.

PaymentRequest

Zach 亮相了 PaymentRequest, PaymentRequest 是一个开放式, 跨浏览器, 内嵌到网站, 安全快速, 随时可以被使用的, 可以用于交易的 API.

PaymentRequest API 预取代传统结账流程, 旨在消灭结账表单.

PaymentRequest 不是一个处理器，不是一种新的支付方式, 也不与支付处理机构直接整合, 仅仅是一个 API.

Seamless && Secure

PaymentRequest 有两个目标

1.为不同的安全支付方式提供无缝支持，让电商公司能够轻松地运用各种完全不同的支付方式
2.安全性, 不是HTTPS的验证, 是考虑适用在任何移动或非移动浏览器，设备或平台上，结合未来的标准化技术实现高品质，一流的支付方式
